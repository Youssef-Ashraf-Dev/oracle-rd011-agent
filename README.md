# RD.011 Future Process Model Generator

An agentic pipeline that reads consultant input files (MoM .docx and optional questionnaire .xlsx) and produces a fully written Oracle Finance Cloud RD.011 Future Process Model Word document. All content is generated by the LLM and assembled into a styled .docx using python-docx.

## Prerequisites
- Conda or Miniconda (Recommended)
- Graphviz (optional but recommended for diagram rendering)

## Installation
The project relies on a Conda environment for dependency management (including the SQLite database for session persistence).
```bash
conda env create -f environment.yml
conda activate rd011-env
```

## How to Run (Streamlit UI)
The project now features a user-friendly Streamlit interface. 

1. Double-click the `Start_RD011.bat` file in the root directory.
2. A web browser will automatically open to `http://localhost:8501`.
3. Enter your API keys directly into the **Settings Sidebar** (No need to configure a `.env` file manually!).
4. Upload your documents and click **Generate**.

*Note: You can safely close your browser or refresh the page. Your progress is saved, and you can resume any session by pasting your Thread ID into the "Resume Session" tab.*

## LLM Configuration & API Keys
The system uses `config.py` (`CAPABILITY_MAP`) to route tasks intelligently to optimize costs and performance:
- **Large Context Ingestion**: OpenRouter (`google/gemini-2.5-flash-lite`)
- **Reasoning/Planning**: OpenRouter (`meta-llama/llama-3.3-70b-instruct`)
- **Fast Generation**: Groq (`llama-3.3-70b-versatile` - Free Tier)

Required API Keys (Entered via UI):
- `OPENROUTER_API_KEY`
- `GROQ_API_KEY`

## Optional RAG knowledge base
RAG is gated by `RAG_ENABLED` in `config.py` and uses Chroma locally.

Embeddings default to a local model (`sentence-transformers/all-MiniLM-L6-v2`) to avoid API rate limits during ingest.
You can override with env vars in `.env`:
- `EMBEDDING_PROVIDER` (`local` or `google`)
- `EMBEDDING_MODEL`
- `EMBEDDING_DEVICE` (`cpu` or `cuda`)
- `RAG_COLLECTION_NAME` (optional; defaults to `rd011_examples_local` for local embeddings)
- `RAG_TRACE_RETRIEVAL` (optional; when true, logs every retrieval result per call)
- `RAG_TRACE_PREVIEW_CHARS` (optional preview length for trace logs)

To build the knowledge base:
1. Place example RD.011 .docx files in `rag/examples/`.
2. Run:
```bash
python -m rag.ingest --docs-dir rag/examples
```

If `RAG_ENABLED=true` but the knowledge base is not built, the agent **fails fast** with a clear error.
If `RAG_ENABLED=false`, the agent runs without RAG context.

For a deeper RAG overview, see `RAG_SOLUTION_DESIGN.md`.

## Usage
Single MoM:
```bash
python main.py --mom samples/inputs_gosaibi_fin_2022/AP20_Formatted.docx
```

Multiple MoMs and questionnaires:
```bash
python main.py \
  --mom samples/inputs_gosaibi_fin_2022/AP20_Formatted.docx \
  --mom samples/inputs_gosaibi_fin_2022/Dec_19_2022_GL_Analysis_MOM_Presentation_Ready.docx \
  --questionnaire "samples/inputs_ssem_fin_2024/AP Questionnaire with Answers.xlsx"
```

Developer utilities live under `tools/` (Python) and `scripts/` (PowerShell).

Resume an interrupted run:
```bash
python main.py --resume --thread-id abc12345
```

## Approval workflow
After planning, the agent presents a document plan and issue list. Type `APPROVE` to proceed or provide feedback text to revise the plan.

## Architecture summary
The pipeline is a LangGraph StateGraph with SQLite checkpointing.

| Node | Task type | Purpose |
| --- | --- | --- |
| ingest | - | Parse .docx and .xlsx inputs |
| extract | LARGE_CONTEXT | Extract structured facts |
| plan | REASONING | Build DocumentPlan |
| detect_issues | REASONING | Contradictions and missing fields |
| present_plan | - | CLI summary for approval |
| await_approval | - | Human-in-the-loop gate |
| update_plan | REASONING | Apply feedback |
| generate_intro | GENERATION | Intro and enterprise context |
| generate_section | GENERATION | Per-process content |
| render_diagrams | - | Graphviz DOT to PNG |
| assemble_document | - | Build final .docx |

LLM routing is configured in `config.py` and enforced through `llm/router.py`.

## Diagram rendering
The generator emits Graphviz DOT in `SectionContent.diagram_code`.
- Primary: Graphviz dot renders PNG.
- Fallback: a Pillow swimlane image is generated if dot is unavailable.
